Exploratory Data Analysis using Pandas Profiling

2020-08-04

Exploratory Data Analysis (EDA) is an essential step in any data science project. This step allows us to remove unuseful/incorrect data, understand the relationships within the data, and provides context required to develop models that will ultimately allow for accurte interpretation of results. EDA can be a time consuming process and this step will have to be revisited if shortcuts are taken. However, pandas-profiling is an extremely powerful python package that can help with this process.

Herein we will look at an example and the few short lines required to generate loads of exploration in a cleanly formated report.

Here is a link to the pandas-profiling project page for more information on the package.

These quick calls on the dataframe gave us a bit of information that is helpful moving forward. We know the how many rows and columns are present, which is of particular interest since a limitation of the pandas-profiling package is the amount of time and memory it requires to run, particularly on large datasets. We also know the column names and their type. With this information, we know which methods the describe call would help us with and which columns we should handle as categories, however, this would require more lines of code! Finally, the sample call gives us a better idea of what the data will actually look like which is important for making decisions about how to cleaning, and later model, the data.

This is all freat information to have and could guide us as we start EDA. There are still many things we need to learn about the data though. For example: is there duplicate data; does the data contain duplicates or missing values; how do the variables relate to one another? Each of these questions dictate how we will handle cleaning, further exploration, and modeling the data to make the best predictions possible. They also require a lot more code to answer! This is where the magic of pandas profiling comes in. In one line of code we can generate a report that answers all of these questions, and more, so we can make efficient use of our EDA time.

profile = ProfileReport(df, title="Pandas Profiling Report")



The pandas profiling package offers a huge amount of information with very little effort. But, there’s more than we’ve looked at here! Some things to try out include: an even deeper dive into your data by using explorative, create widgets or iframes with the information in your report, or deal with large datasets using minimal.

profile = ProfileReport(df, title="Pandas Profiling Report", explorative=True)  #deeper dive into the data
profile.to_widgets() #creates a widget containing all of the report information
profile.to_notebook_iframe() #creates an iframe containing all of the report information
profile = ProfileReport(huge_df, title="Pandas Profiling Report", minimal=True)
